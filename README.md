1.Daily Interaction with cloud models -Infrastructure as a Service(IaaS):This is the foundation level.When an company uses Amazon EC2,they are renting virtual hardware. They have 
control over the operating system and the software stack,but the cloud provider owns and maintains the physical servers and networking.

-Platform as service(PaaS):As a developer,I use platforms like GitHub pages to  host a website,these services provide the environment for me to deploy my
code without worrying about the underlying operating system or server maintenance.

-Software as a service(SaaS):These are app we use every day where we don't manage anything but the data we put in.Examples include Google Drive for storage
or Netflix.You just log in and use the service;Google or Netflix handle the server,the code,and the updates.

2.Research on Major Cloud Provider
(i)Amazon Web Services(AWS):AWS is currently the market leader in cloud computing.It offers a massive ecosystem of over  200 fully featured services.
Its primary strength lie in Amazon EC2(Scalable computing) and Amazon S3(object storage).AWS is known for its "pay-as-you-go" pricing model and its massive
global infrastructure,which allows businesses to deploy applications in minutes across multiple continents.

(ii)Goggle Cloud Platform(GCP):GCP is highly regarded for its expertise in data analytics and machine learning.Its standout services include BigQuery
for data warehousing and Google Kubernates Engine(GKE) for managing containerized apps.Since Google uses the same infrastructure for its search engine
and YouTube,GCP offers incredible speed and high-performance networking capabilities that are often more cost-effective for data-heavy projects.
